# Plan Template

## 1. 需求文档

记录 owner 的明确需求、完成标准和不做范围。

## 2. 当前事实

只写当前仓库、命令输出和 owner 输入确认的事实。

## 3. 失败测试

列出哪些情况说明任务失败。

## 4. 目标

写可验收的终局，不写“后续优化”。

## 5. 不做范围

明确本轮不处理的边界。

## 6. 设计

说明文件职责、数据流、接口边界和验证方式。

## 7. 实施任务

- [ ] T001 ...

## 8. 验证计划

```powershell
npm.cmd run verify
```

## 9. 收口

记录实际验证命令、结果和剩余风险。
